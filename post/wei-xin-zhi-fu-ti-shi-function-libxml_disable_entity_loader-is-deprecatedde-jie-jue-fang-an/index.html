微信支付提示Function libxml_disable_entity_loader() is deprecated！的解决方案

· 2026-04-29 · # Fastadmin

想象一下，你的 PHP 程序在解析微信支付发来的 XML 数据时，需要一个 "安全开关" 来防止黑客通过 XML 文件攻击你的服务器。这个开关就是 libxml_disable_entity_loader() 函数。
现在，PHP 升级到 8.0 以后，官方认为这个 "开关" 的设计太老了，直接把它给 "淘汰" 了。当你的程序（尤其是微信支付SDK）还在用这个旧开关时，系统就会报出这个弃用警告。

🛠️ 一、解决方案：换上新"开关"，安全又兼容

既然旧开关不让用了，就得换上新的 PHP 8.0 认可的安全设置。主要需要修改以下文件：

📁 修改文件：vendor/overtrue/wechat/src/Kernel/Support/XML.php 中的 parse 方法。

💡 修改思路：根据 PHP 版本，选择不同的 XML 解析方式，确保无论版本高低，都使用最安全的方式。

方案一：统一使用最安全的新方法 ⭐ 推荐

最省心、最安全的方法。不再区分 PHP 版本，统一使用 PHP 官方推荐的安全选项来解析 XML。

public static function parse($xml)
{
    // 核心：使用 LIBXML_NONET 等安全选项，彻底替代旧函数
    $options = LIBXML_NONET | LIBXML_NOCDATA | LIBXML_NOBLANKS;
    $xmlObject = simplexml_load_string($xml, 'SimpleXMLElement', $options);

    if ($xmlObject === false) {
        throw new \EasyWeChat\Kernel\Exceptions\InvalidArgumentException('Invalid XML string.');
    }
    return self::normalize($xmlObject);
}

👍 好处：代码简洁，一劳永逸，安全有保障。LIBXML_NONET 这个选项能禁止 XML 解析时访问网络，是防御黑客攻击的关键。

方案二：保留版本判断，但优化新版本逻辑

如果你担心兼容性，可以保留对 PHP 版本的判断，但优化 PHP 8.0 以上的处理逻辑。

public static function parse($xml)
{
    if (\PHP_VERSION_ID < 80000) {
        // PHP < 8.0：使用旧方法
        $backup = libxml_disable_entity_loader(true);
        $result = self::normalize(simplexml_load_string(
            self::sanitize($xml),
            'SimpleXMLElement',
            LIBXML_COMPACT | LIBXML_NOCDATA | LIBXML_NOBLANKS
        ));
        libxml_disable_entity_loader($backup);
    } else {
        // PHP >= 8.0：使用新方法
        $result = json_decode(json_encode(
            simplexml_load_string($xml, 'SimpleXMLElement', LIBXML_NOCDATA),
            JSON_UNESCAPED_UNICODE
        ), true);
    }

    return $result;
}

👍 好处：对旧环境兼容性更好，但代码稍复杂。

⚠️ 二、修改后必做事项

  1. 🧹 清除缓存：修改了 vendor 目录下的文件后，一定要清除项目的缓存（比如 Opcache、框架缓存），否则新代码可能不生效。
  2. 🧪 全面测试：改完后，必须完整地走一遍微信支付流程，特别是支付成功后的 "异步回调通知"，确保支付状态能正常更新到你的数据库。

🚧 三、你可能还会遇到的"连环坑"

问题 原因 解决方案
📭 支付回调收不到数据 PHP 7.0+ 使用 $GLOBALS['HTTP_RAW_POST_DATA'] 可能取不到数据 改成 file_get_contents("php://input")
🔄 突然报 302 Found 错误 微信服务器 IP 变更，但 hosts 文件仍指向旧 IP 更新 hosts 文件或检查 DNS 解析
🔒 SSL 证书验证错误 测试环境证书问题（curl 错误码 60） 临时关闭 CURLOPT_SSL_VERIFYPEER，但线上环境勿用
📱 小程序订阅消息报错 48001 用错了接口地址 使用专门的订阅消息接口

💎 四、总结

解决 libxml_disable_entity_loader() is deprecated 警告，就是把微信支付 SDK 里解析 XML 的旧安全设置，换成 PHP 8.0 以上版本认可的新安全设置。

最直接的办法就是修改 XML.php 文件，使用 LIBXML_NONET 等选项。这就像给程序做一次 "安全升级"，既消除了烦人的警告，也让你的应用在面对潜在的 XML 攻击时更坚固。
