Bootstrap Table中不转义Html标签的解决方案

· 2026-04-29 · # Fastadmin

在 Bootstrap Table 中直接显示 HTML 标签内容是一个常见需求，比如希望 <br> 标签能实际换行而非显示为文本。问题的核心通常在于一个关键的配置参数。下面梳理解决方案和注意事项。

🔧 一、核心解决方案：关闭 escape 选项

Bootstrap Table 提供了一个专门的列选项（escape）来控制是否对 HTML 内容进行转义：

参数值 效果
true（默认） 原样输出 HTML 标签（如 < 显示为 &lt;）
false 正常解析并渲染 HTML 标签

设置方法

设置方式 代码示例 说明
HTML data-* 属性 data-escape="false" 直接写在表头 <th> 标签中，用于特定列
JavaScript 配置 escape: false 在初始化表格的 JS 代码中设置，全局生效

典型配置示例

$('#yourTableId').bootstrapTable({
    url: $.fn.bootstrapTable.defaults.extend.index_url,
    pk: "id",
    escape: false, // 表格内容将不会被转义
    columns: []
});

⚠️ 二、安全警告与最佳实践

在关闭转义功能前，必须高度重视由此带来的安全风险。

风险来源

如果表格中显示的数据来自用户输入或不可信的第三方，直接关闭转义可能导致 XSS（跨站脚本）攻击。攻击者可以注入恶意脚本：

<script>alert('XSS')</script>

这些脚本会在其他用户的浏览器中执行，可能导致 Cookie 等信息被盗。

安全准则

数据类型 建议
✅ 可信数据 仅在显示你完全可控、绝对可信的静态数据时，才考虑关闭转义
❌ 不可信数据 对于任何来自用户或外部的不可信数据，务必保持转义开启（默认 true）

💡 三、替代方案与思路

如果数据不可信但又需要复杂的格式，可以考虑以下更安全的方案：

方案一：使用安全的格式化工具

利用 Bootstrap Table 内置的 formatter 函数，使用安全的 DOM 操作方法构建内容：

{
    field: 'content',
    title: '内容',
    formatter: function(value, row, index) {
        // 使用 document.createTextNode 安全创建文本节点
        // 或使用经过严格验证的逻辑来构建内容
        return value;
    }
}

方案二：在数据源头处理

在后端生成数据时，就将需要换行的地方直接转换为安全的 HTML 实体或使用特定的标记语言。

💎 四、总结与建议

解决 Bootstrap Table 中 HTML 标签不解析的问题，关键在于正确设置 escape: false 参数。

推荐决策流程

flowchart TD
    A[需要显示HTML内容] --> B{数据来源是否绝对可信？}
    B -- 是 --> C[设置 escape: false]
    B -- 否 --> D[保持 escape: true<br>（默认值）]
    D --> E[寻求替代方案<br>（如使用formatter函数）]
