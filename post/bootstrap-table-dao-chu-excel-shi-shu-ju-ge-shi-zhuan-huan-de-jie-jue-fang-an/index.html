Bootstrap Table导出Excel时数据格式转换的解决方案

· 2026-04-29 · # Fastadmin

在 Fastadmin 框架中，Bootstrap Table 导出 Excel 时会遇到数据格式被 Excel 自动转换的问题，比如数字变成科学计数法、日期格式错乱等。下面梳理常见的格式问题及其解决方案。

💡 一、数字格式问题与修复

数字格式问题最常见，特别是长数字（如身份证号、银行卡号）被转换为科学计数法。

核心解决方法

使用 onMsoNumberFormat 回调函数，通过设置 Excel 的数字格式代码来强制指定单元格格式：

格式类型 代码 适用场景
文本格式 \@ 需要完整显示的长数字
数字格式 0 无小数位数字
小数格式 0\.000 三位小数
百分比 0% 百分比显示

具体配置示例

exportOptions: {
    mso: {
        onMsoNumberFormat: function (cell, row, col) {
            // 全部设为文本格式
            return '\\@';

            // 针对特定列设置格式（第6列设为两位小数）
            // return (col == 5) ? '0\\.00' : '';

            // 或者所有数字列设为文本格式
            // return !isNaN($(cell).text()) ? '\\@' : '';
        }
    }
}

⏰ 二、日期与特殊字符处理

日期格式（如 5/10、5-10）容易被 Excel 自动识别为日期。

解决方案

使用 onCellHtmlData 回调全局处理特定内容：

exportOptions: {
    onCellHtmlData: function(cell, row, col, data) {
        if (typeof data === 'string' && (data.includes('/') || data.includes('-'))) {
            return "'" + data; // 为含斜杠的内容添加单引号
        }
        return data;
    }
}

⚙️ 三、其他实用配置

忽略特定列

ignoreColumn: [0, 'operate']  // 跳过复选框和操作列

自定义文件名

fileName: '文件名_' + Moment().format("YYYY-MM-DD")

导出范围控制

exportDataType: "all"       // 全部数据
exportDataType: "selected"  // 选中行
exportDataType: "basic"     // 当前页

💎 四、总结

解决 Fastadmin 中 Bootstrap Table 导出格式问题的核心是合理使用导出回调函数：

  • 数字格式问题 → 优先使用 onMsoNumberFormat
  • 日期和特殊格式 → 考虑 onCellHtmlData

如果你有特定的格式问题场景，可以进一步描述，提供更具体的配置建议。
